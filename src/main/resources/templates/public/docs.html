Quick Start

Blob Store is the control layer for structured uploads and retrieval.

Use it to define project-based upload modules, enforce validation rules, generate image variants, and expose file APIs to other systems. The rest of the dashboard handles configuration, operations, and observability; this page gives you the fast overview.

Organize by project Create a project once, then group related modules under it so storage paths and admin structure stay predictable.
Control file behavior Each module defines allowed media types, size limits, privacy rules, and image processing requirements.
Serve other apps External systems upload and retrieve files through the API layer instead of using the admin UI directly.

Security Notes

Admin access and API access are intentionally separated so operational access does not get mixed with application integration access.

Admin UI The dashboard uses form login with a local admin account loaded from blobstore.admin.config-file. This is the right place for internal operations access, not for application-to-application traffic.
API authentication Application-facing APIs use JWT validation and can optionally enable Basic Auth for tightly controlled backend integrations. Configure this from the admin dashboard under API Security.
Public vs private files Modules marked with Public Retrieval Allowed can be retrieved anonymously. Private modules require an authenticated token for file download and metadata access.
Recommended token scopes Use blobstore.modules.read, blobstore.modules.write, and blobstore.files.write for the secured routes. Basic Auth should be reserved for backend integrations over trusted transport only.

API Reference

Use these endpoints from other services. Configuration happens in the dashboard; upload and retrieval flows happen through the REST API.

Upload POST /api/files/{moduleCode}
Retrieve GET /api/files/{fileKey}
Image Variant GET /api/files/{fileKey}?type=thumb
Modules GET /api/modules
Payload multipart/form-data with field name "file"; image modules can also enforce an exact original size configured as WIDTHxHEIGHT
Retrieval can stay anonymous only when a module is marked public. Uploads require configured API credentials, and private file reads require authenticated access through JWT or an allowed backend integration client.
